L2: Brand Context Layer

Build the "brain" that assembles the right context for each agent call. No LLM reasoning yet — just data assembly.
M7 · Context Assembler
Pending
  • Brand context builder: BrandProfile + BrandCategoryPolicy + active BrandDirectives → JSON
  • Domain context queries: per-agent summaries (Growth sees ad data, Supply sees inventory)
  • Task context queries: per-SKU deep dive (signals + P&L + history) for a specific trigger
  • Context size budgeting: ensure each layer fits within token limits (~2K + ~8K + ~5K)
M8 · Decision Memory
Pending
  • Design Decision record schema (trigger → context → reasoning → action → outcome)
  • Store every agent decision as a structured record in DB
  • Retrieval: "last 10 decisions on this SKU" and "last 5 decisions by Growth Agent"
  • Pruning: keep recent + high-impact, summarise older decisions
M9 · Morning Briefing (v1)
Pending
  • Daily scan job: query ThrottleSignals for anomalies (ROAS drops, OOS, margin breaches)
  • Generate briefing via Haiku (template-driven, not free-form)
  • Surface in dashboard — the "what happened overnight" card
  • Email/notification to operator

L3: Agent Runtime

First agent goes live. Starts deterministic (Python functions), earns trust before getting LLM powers.
M10 · Growth Agent (Deterministic)
Pending
  • Base agent interface: observe() → reason() → propose()
  • Agent registry: register agents, route triggers to the right agent
  • Growth Agent: implement playbooks as Python functions
  • OOS response — auto-pause ads
  • Low cover — throttle bids 30–50%
  • High cover — boost aggression
  • ROAS drop — pause underperformers, reallocate budget
  • D2C conversion drop — investigate funnel, diagnose cause
  • Growth Agent output: structured Intent objects (not free text)
M11 · Inter-Agent Coordination
Pending
  • Cross-agent queries: Growth asks Supply "when will this SKU restock?"
  • Shared decision feed: agents can read each other's recent decisions
  • Conflict detection: Growth wants to boost, Supply says OOS → Growth defers
M12 · Approval Workflow
Pending
  • Approval queue in DB (pending decisions awaiting operator response)
  • Surface in dashboard right panel (already mocked in UI)
  • Approve/reject flow → decision record updated → agent proceeds or backs off
  • Auto-expire: stale approvals expire after 24h with notification

L4: Policy & Governance

Hard guardrails that agents cannot override. Every action passes through this gate.
M13 · Policy Engine
Pending
  • Rule evaluator: check BrandProfile + BrandCategoryPolicy + BrandDirectives
  • Policy types: hard constraint, approval routing, auto-execute, time window, escalation
  • Kill switch: halt all automated actions for a brand instantly
  • Audit trail: every decision shows trigger → reasoning → policy check → outcome
M14 · Operator Controls
Pending
  • Directive management UI: create/edit/revoke directives from dashboard
  • Policy dashboard: see all active policies, directives, and their status
  • Override history: "who changed what and when"

L5: LLM-Powered Reasoning

Swap deterministic playbooks for LLM reasoning. Playbooks become system prompt context, not code.
M15 · LLM Agent Mode
Pending
  • Playbook .md files → system prompt injection (the "skill file" approach)
  • Model router: Haiku for routine checks, Sonnet for complex multi-signal decisions
  • Confidence scoring: agent reports confidence level, low confidence → escalate
  • Reasoning trace: agent explains its thinking (stored in Decision record)
M16 · Full Agent Roster
Pending
  • Supply Agent: stockout prediction, reorder triggers, FC rebalancing
  • Category Agent: competitive pricing, discount strategy, BSR tracking
  • Creative Agent: creative fatigue detection, brief generation
  • Each agent follows the same interface, plugs into the same policy engine
